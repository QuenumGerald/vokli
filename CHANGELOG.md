# Changelog

All notable changes will be documented here.

## 0.4.0 - 2026-07-17

- Add `vapiKnowledge()` and the `vokli.knowledge` validation, status, and
  incremental synchronization API.
- Add SHA-256 document tracking and atomic `.vokli/state.json` persistence.
- Add knowledge-aware receptionist prompt rules and the Knowledge Base guide.

## 0.2.0 - 2026-07-17

- Add the reusable receptionist definition and structured runtime validation.
- Add deterministic prompt and structured-output generation.
- Add pure generation of configurable local Vapi resource drafts.
- Expand the SDK with `validate()` and `generate()` and add a garage example.

## 0.1.0 - 2026-07-17

- Add the Phase 1 TypeScript monorepo foundation.
- Add provider-independent definitions and agent validation.
- Add minimal Vapi, knowledge, testing, and SDK public boundaries.
